# 竞品调研与对比（对应 md 第 4 节 & 实验报告第 5 节）

> 说明：以下为调研框架与对比维度，具体数据需团队补充引用来源后填入实验报告。

## 参考系统

| 系统 | 定位 | 关键特征 |
|---|---|---|
| **DeepAudit** | 面向 Web 的代码漏洞挖掘智能体系统 | 多智能体协作 + 自动化沙箱 PoC 验证 |
| **AgentStalker** | 首个以 LLM 智能体行为为对象的端到端安全审计框架 | 静态建模→意图假设→沙箱动态验证→证据聚合 |
| **OpenSecurity** | AI 驱动的多层安全测试 Agent 平台 | LLM 驱动多工具，完成端到端安全测试 |
| **ESAA-Security** | 新兴开源架构的安全审计系统 | 安全审计行为可确定、可水线，输出格式化清单 |
| **Sandyaa** | 基于智能体的审计工具 | 可递归深入挖掘，直到验证出真实漏洞 |

## 对比维度（建议在报告中做成表格打勾）

- 是否融合 SAST 工具 + LLM 语义审计
- 是否具备独立验证智能体（降低误报）
- 是否有动态 PoC / 沙箱验证
- 证据链是否可追溯
- 结果是否可复现（保存 prompt/模型输出/参数）
- 报告是否结构化、支持多格式
- 支持语言 / 框架覆盖面
- 是否开源、可扩展

## AuditAgentX 的差异化定位

对照上表，本系统主打 **「多源融合 + 双智能体交叉验证 + 可追溯证据链 + 可复现」** 组合，
避免"直接调用大模型审计"导致的高误报、无证据、不可复现三大痛点。
